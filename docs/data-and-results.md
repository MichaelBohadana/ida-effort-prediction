# Data and results files

This page describes the datasets the study uses and the result files kept in
`results/`. It also maps each committed CSV to the table or figure it backs, so
you can check a number without rerunning anything.

## The datasets

A labeled row pairs a start state and a bound with its effort label, the number
of nodes the bounded pass expands. For each state the scripts generate one row
per offset, where the bound is `h(state) + offset`. Rows that hit the node cap
are censored: they are dropped and counted.

There are five datasets, in three roles. The 12-pancake pair is the main
comparison, the 10-pancake pair supports the depth study, and the 15-puzzle is
the generalization study.

| Dataset          | States | Offsets | Node cap        | Rows  | Censored |
|------------------|-------:|---------|-----------------|------:|---------:|
| 12-pancake, GAP  | 1,500  | 0--3    | 2e7             | 6,000 | 0        |
| 12-pancake, RAND | 1,500  | 0--1    | 2e6             | 3,000 | 0        |
| 10-pancake, GAP  | 1,000  | 0--4    | 2e7             | 5,000 | 0        |
| 10-pancake, RAND | 1,000  | 0--3    | 2e7             | 4,000 | 0        |
| 15-puzzle, MD    | 1,500  | 0--10   | 2e7             | 9,000 | 16       |

Notes:

- Start states are sampled uniformly at random (solvable states for the
  15-puzzle), not by random walks from the goal. This removes a confound in
  KRE's per-state error.
- The two pancake heuristics are labeled on the same 1,500 states.
- On the 15-puzzle only even offsets are used, because Manhattan distance
  changes by exactly one per move, so odd offsets add no fertile nodes.
- The label datasets themselves are large and regenerable, so they are not
  committed. The commands in [workflow.md](workflow.md) rebuild them.

## What is committed and what is not

Only the small summary CSVs are committed, so the reported numbers can be read
directly. Everything regenerable is left out and rebuilt by the pipeline. The
`.gitignore` enforces this.

Committed: the summary CSVs listed below, plus `results/.gitkeep`.

Not committed (regenerated by the pipeline): the raw label datasets
(`*labels*.csv`), the sampled distributions (`*.npy`), the figures (`*.png`,
`*.pdf`), and the run logs (`*.log`).

## Which file backs which result

| Result in the report          | Committed CSV                                            |
|-------------------------------|----------------------------------------------------------|
| Headline comparison           | `pancake_eval_gap.csv`, `pancake_eval_rand.csv`          |
| Equal-information ablation    | `pancake_eval_{gap,rand}_minimal.csv`                    |
| Depth study                   | `pancake_eval_{gap,rand}_n10*.csv` and `_by_offset.csv`  |
| Extrapolation                 | `extrapolation_test.csv`, `extrapolation_tiers.csv`      |
| Tail errors                   | `tail_errors.csv`                                        |
| Learning curves               | `learning_curve10.csv` (`learning_curve.csv` is the pilot)|
| Offline costs                 | `cost_ledger.csv`                                        |
| Prediction latency            | `prediction_latency.csv`                                 |
| Probe tier                    | `probe_eval.csv`, `probe_no_fertile.csv`                 |
| Feature importance            | `mlp_importance_gap.csv`, `mlp_importance_rand.csv`      |
| Split robustness              | `robustness_seeds10.csv` (`robustness_seeds.csv` is the 5-seed pilot) |
| 15-puzzle generalization      | `tiles_generalization_eval.csv`                          |

The `eval_table.csv` file is the earlier walk-based 15-puzzle pilot, kept for
reference. The paper's 15-puzzle numbers come from the uniform-state study in
`tiles_generalization_eval.csv`.
